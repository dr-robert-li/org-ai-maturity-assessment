# Quarterly Red-Teaming Checklist

> **Purpose**: Establish a regular cadence for testing AI system safety, reliability, and compliance. Run this quarterly to catch drift, surface risks, and keep governance current.
>
> **Source**: Adapted from [Emerging Practices in Team Building for an Agentic Internet](https://drli.blog/posts/emerging-practices-3t-team/)

---

## Month 1: Assessment and Planning

- [ ] Review prior quarter's incidents and near-misses
- [ ] Update risk assessment to reflect new AI deployments
- [ ] Schedule red-teaming exercises for Month 2
- [ ] Audit marketing data privacy practices
- [ ] Review consent management systems
- [ ] Test privacy request fulfilment procedures

## Month 2: Active Testing

- [ ] Conduct red-teaming exercises (prompt injection, data poisoning, adversarial examples)
- [ ] Test model performance and drift detection
- [ ] Evaluate bias and fairness across demographics
- [ ] Conduct information integrity assessment
- [ ] Run incident response tabletop exercise
- [ ] Review third-party and vendor compliance

## Month 3: Review and Remediation

- [ ] Analyse results for patterns and recurring themes
- [ ] Document findings and remediation steps with owners and due dates
- [ ] Update protocols based on learnings
- [ ] Refine containment strategies
- [ ] Prepare quarterly report (metrics, trends) for leadership
- [ ] Plan next quarter's testing focus areas

## Continuous Activities

- [ ] Monitor AI system performance metrics daily
- [ ] Log incidents in tracking system immediately
- [ ] Weekly privacy rights request review
- [ ] Monthly employee training on AI safety
- [ ] Update documentation as systems change
