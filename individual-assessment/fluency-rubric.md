# Individual AI Fluency Rubric

> **Purpose**: Score an individual (or team) across five dimensions on a 1–4 scale. Use alongside the [Fluency Checklist](fluency-checklist.md) for a complete picture. This rubric is calibrated for business operations users working with Claude Desktop but can be adapted for any role or platform.

---

## Rubric

| Dimension | 1 — Novice | 2 — Developing | 3 — Proficient | 4 — Fluent |
|-----------|-----------|---------------|----------------|-----------|
| **Prompting Quality** | Vague or one-line prompts, accepts first response | Adds some context, occasionally iterates | Clear role/task/format, iterates reliably | Consistent structured prompts, coaches others |
| **Output Judgement** | Accepts outputs uncritically | Occasionally checks facts | Routinely verifies and edits outputs | Actively stress-tests, catches edge cases |
| **Domain Application** | Generic use only (summarise emails) | Has mapped Claude to 1–2 domain tasks | Uses Claude across multiple domain workflows | Claude is embedded in regular work rhythms |
| **Data and Safety Awareness** | No awareness of data risks | Aware but not consistently applying | Applies data hygiene consistently | Has documented or shared a guideline with the team |
| **Team Leverage** | Solo user only | Shares outputs occasionally | Team has shared prompts or conventions | Team is building repeatable Claude-assisted workflows |

---

## Scoring Guide

| Total Score | Stage | Interpretation |
|:-----------:|-------|----------------|
| **16–20** | **Fluent** | Ready for advanced use cases, API/integration conversations, and internal champion role |
| **11–15** | **Proficient** | Solid foundation with targeted skill gaps to close |
| **6–10** | **Developing** | Consistent daily use needed, structured learning recommended |
| **5** | **Novice** | Needs onboarding before assessing further |

## Generating Recommendations from Scores

### Score of 1–2 in any dimension → Priority gap

These are blockers that should be addressed in the first 30 days.

| Dimension Gap | Recommended Actions |
|---------------|---------------------|
| Prompting Quality | Provide 3–5 worked examples of structured prompts for their domain. Run a 30-minute coaching session on role/task/context/format framing. |
| Output Judgement | Introduce a simple verification habit: "Before sharing any Claude output, check one fact and one number." Build a lightweight eval checklist for their top 2 use cases. |
| Domain Application | Map Claude to 3 specific recurring tasks in their workflow. Demonstrate each one live. Set a goal of using Claude for at least one of these tasks daily for two weeks. |
| Data and Safety Awareness | Draft a one-page data handling guideline covering what can and cannot be pasted. Review with the person and their team. |
| Team Leverage | Start a shared Slack/Teams channel for prompt sharing. Schedule a 30-minute show-and-tell. Identify one other user to pair with. |

### Score of 3 → Strengthening opportunity

These are areas where the person is competent but not yet creating leverage for the team.

| Dimension at 3 | Recommended Actions |
|-----------------|---------------------|
| Prompting Quality | Introduce multi-turn conversation habits and prompt versioning. Have them document their best prompts. |
| Output Judgement | Build output rubrics for their top 2–3 use cases. Introduce informal "test cases" alongside saved prompts. |
| Domain Application | Embed Claude into one recurring workflow cadence (e.g., weekly pipeline review, QBR prep). |
| Data and Safety Awareness | Formalise their informal practices into a team guideline document. |
| Team Leverage | Transition from ad-hoc sharing to a shared prompt library. Explore Claude Projects for team context. |

### Score of 4 → Leverage and scale

These are strengths to amplify.

| Dimension at 4 | Recommended Actions |
|-----------------|---------------------|
| Any dimension at 4 | This person is a candidate for internal champion / power user role. Have them lead show-and-tells, mentor others, and contribute to organisational adoption strategy. |

## Example: Completed Assessment

| Dimension | Score | Evidence |
|-----------|:-----:|----------|
| Prompting Quality | 3 | Clear prompts with role/task framing. Single-turn habit — not yet using multi-turn or prompt versioning. |
| Output Judgement | 3 | Reviews outputs before sharing. No structured eval process. Has caught errors manually. |
| Domain Application | 3 | Uses Claude for document drafting, data interpretation, and template creation. Not yet embedded in recurring cadences. |
| Data and Safety Awareness | 2 | Loosely understands risks. No formal policy. Pastes data without consistent guidelines. |
| Team Leverage | 1 | Solo user. Does not share prompts or teach others. No team conventions. |
| **Total** | **12/20** | **Proficient** — strong individual capability, team and governance layer underdeveloped. |

### Recommendations from this example:

1. **Priority 1** (Data gap, Team gap): Draft data handling guideline, start shared channel, nominate champion
2. **Priority 2**: Build shared prompt library, introduce multi-turn habits, activate unused integrations
3. **Priority 3**: Introduce output rubrics and informal eval practice
4. **Priority 4**: Embed Claude into one recurring RevOps cadence, explore Claude Projects
